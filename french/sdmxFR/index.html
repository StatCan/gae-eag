    SDMX

    Qu’est-ce que SDMX?

    SDMX, qui signifie Statistical Data and Metadata eXchange (échange de données et de métadonnées statistiques), est une initiative internationale qui vise à normaliser et à moderniser (« industrialiser ») les mécanismes et processus d’échange de données et de métadonnées statistiques entre les organisations internationales et leurs pays membres.

    En résumé, SDMX est une nouvelle technologie qui simplifie la transmission des données statistiques. SDMX est une norme et une technologie permettant de styliser et de transmettre des données à l’aide d’un langage machine à machine. C’est :

    • un modèle logique pour décrire et structurer le contenu;
    • une norme pour la communication automatique;
    • une technologie au service de la normalisation.

    SDMX à StatCan

    Actuellement, les renseignements de profil de recensement sont disponibles à tous les niveaux géographiques au moyen d’une interface de programmation d’applications (API) de SDMX. Dans un avenir rapproché, tous les produits CODAR seront disponibles au moyen d’une API de SDMX. Un petit exemple est inclus ci-dessous.

    En savoir davantage sur l’API de SMDX du Profil du Recensement de 2021


    L’explorateur de données de SDMX permet aux utilisateurs d’interroger et de visualiser les résultats dans leur navigateur. Les nouveaux utilisateurs de SDMX y voient généralement une utilité lorsqu’ils tentent de comprendre les données avant de se connecter au moyen de l’API.

    Exemple d’API :

    STC Census Profile SDMX Python API
    import pandas as pd
import pandasdmx as sdmx
info = {
  "id": "STC_CP",
  "url": "https://api.statcan.gc.ca/census-recensement/profile/sdmx/rest",
  "name": "STC Census Profiles"
}
sdmx.add_source(info, id=None, override=True)
stc = sdmx.Request('STC_CP')
resp = stc.data('df_pr')
#df_pr- Canada, provinces and territories dataflow
data = resp.to_pandas()
#returns a series with a multi-index, "2-d series; 3-d dataframe"
data
    Le code ci-dessus renvoie ce cadre de données multi-indices: Après conditionnement de la colonne Ref_Area, vous pouvez ensuite la joindre à un cadre de données spatiales ou à une couche géographique.

    Code de flux de données Nom du flux de données
    DF_PR * Canada, provinces et territoires (*utilisé dans l’exemple ci-dessus)
    DF_ADA aire de diffusion agrégée
    DF_CD divisions de recensement
    DF_CMACA régions métropolitaines de recensement et agglomérations de recensement
    DF_CSD subdivisions de recensement (municipalités)
    DF_CT secteurs de recensement
    DF_DA aires de diffusion
    DF_DPL localités désignées
    DF_ER régions économiques
    DF_FED circonscriptions électorales fédérales (Ordonnance de représentation de 2013)
    DF_POPCNTR centres de population
