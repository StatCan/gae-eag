    Une introduction sommaire à l’analyse géospatiale

    L’analyse géospatiale s’intéresse à la localisation quantitative de caractéristiques importantes, appelées phénomènes.

    Les données géospatiales sont des données concernant des phénomènes qui ont un emplacement sur la surface de la terre. L’emplacement peut être statique (p. ex. emplacement d’une route, catastrophe, citoyens vivant dans la pauvreté) ou dynamique (p. ex. véhicule ou piéton en mouvement, propagation de la COVID-19).

    Un SIG (système d’information géospatiale) nous aide à analyser les relations et les profils spatiaux parmi ces phénomènes. Il est particulièrement utile pour la visualisation des données spatiales et la présentation des résultats d’analyse.

    Les SIG sont composés d’éléments matériels et logiciels. En raison d’importants volumes de données spatiales et de la nécessité d’afficher les valeurs à la volée, le matériel du SIG dispose souvent de grandes capacités de stockage, d’une vitesse de calcul rapide et d’une mémoire puissante. L’ADS a été conçue pour répondre aux besoins de ces utilisateurs.

    À propos des données spatiales

    Les données spatiales sont des types de données (fichiers, bases de données, services Web) qui codent l’information géographique destinée à être utilisée dans des applications fondées sur l’emplacement. Il en existe deux types :

    1. Données vectorielles
      • points (coordonnées x, y, (z))
      • lignes (routes, réseaux))
      • polygones (zones).
    2. Données de trame
      • Images ou grilles de valeurs numériques (p. ex. imagerie satellitaire).

    Une base de données spatiales – également appelée base de données géospatiales– est construite pour capturer et stocker les points, les lignes et les polygones.

    Vocabulaire du SIG

    Term Description
    Carte de base Les données du SIG et/ou les images qui constituent le fond de la carte
    Coordonnées Valeurs représentées par les lettres x, y, z [ou m (mesure)], qui définissent une position dans une référence spatiale.
    DCAT, GeoDCAT AP Norme de métadonnées permettant de rechercher des ensembles de données spatiales, des séries d’ensembles de données et des services dans les catalogues généraux de données.
    OT, observation de la Terre Information sur la planète Terre recueillie par des technologies de télédétection ainsi que par des techniques au sol (p. ex. imagerie par satellite ou par drone).
    ETL Procédure consistant à copier des données de sources vers un système de destination qui représente les données différemment de la source ou dans un contexte différent de la source.
    Généralisation Méthode utilisée dans le SIG pour réduire le détail des données (pour des raisons de confidentialité et/ou de visualisation).
    Géocoder Le processus d’identification d’un emplacement par un ou plusieurs attributs d’une couche.
    HNAP HNAP-Canada est un profil du profil nord-américain harmonisé qui est lui-même un profil de la norme ISO19139 (ISO 19115*).
    Couche Données thématiques décrites et stockées dans une bibliothèque de cartes. Les couches font fonction de transparents qui peuvent être superposés pour la visualisation ou l’analyse spatiale.
    OGC Open Geospatial Consortium – champion du code source ouvert; normes ouvertes.
    PostgreSQL Système de base de données relationnel-objet de source ouverte; géré par Azure.
    Fichier forme Format de stockage de données vectorielles permettant de stocker l’emplacement, la forme et les attributs. Il est stocké dans un ensemble de quatre fichiers connexes et contient une classe de caractéristiques. Il faut éviter les fichiers formes en production.
    WMS Web Map Service; spécifications permettant aux clients Web d’accéder de façon uniforme aux cartes rendues par les serveurs cartographiques sur Internet.
    WFS Web Feature Service; spécifications qui normalisent la manipulation des données et l’affichage des cartes sur Internet.
