    ArcGIS Pro on DAaaS GEO

    Votre licence ArcGIS Pro 

    Les licences ArcGIS Pro sont gérées par l’équipe StatCan GeoHelp.

    Il s’agit de la même licence que celle que vous utilisez pour accéder à ArcGIS Pro sur AVD et ArcGIS Online.

    l'ADS ne peut pas vous fournir de licence.  

    Veuillez contacter GeoHelp pour obtenir de l’aide : geohelp@statcan.gc.ca

    Connectez votre licence ArcGIS Pro (utilisateur interne STC)

    1. Dans le menu Paramètres, sélectionnez "Licences" dans la liste sur le côté gauche. Il est normal que le gestionnaire de licences prenne plusieurs minutes pour s'initialiser.

    2. Cliquez sur Configurer vos options de licence

    3. Dans le menu déroulant Type de licence, sélectionnez l'option Utilisation simultanée.

    4. Dans le champ Gestionnaire de licences, entrez le nom d'hôte : Stmcapp-200.statcan.ca

    5. Cliquez sur le bouton "Ok" pour enregistrer vos modifications.

    6.Redémarrez ArcGIS Pro pour activer les nouveaux paramètres de licence.

    Une fois que vous êtes connecté au serveur de licences simultanées, vous devriez pouvoir voir les licences pour le logiciel ArcGIS Pro et toutes les extensions qui y sont incluses.

    Connecter Pro au portail ArcGIS Enterprise

    Une connexion au portail ArcGIS Enterprise de DAaaS GEO doit être établie pour pouvoir publier depuis ArcGIS Pro vers le portail. Procédez comme suit:

    1. a: Dans l’écran d’accueil d’ArcGIS Pro, sélectionnez Paramètres, puis Portails

      OU

      b : Dans un projet, cliquez avec le bouton droit sur votre nom en haut à droite de la fenêtre et sélectionnez : Gérer les portails

    2. Sélectionnez Ajouter un portail et entrez l’URL suivant: https://geoanalytics.cloud.statcan.ca/portal/

    3. La connexion sera alors répertoriée. Sélectionnez le bouton plus '...', puis sélectionnez Se connecter.

      Nous vous suggérons également, une fois la connexion terminée, de sélectionner le portail Définir comme Actif dans le menu plus... déroulant.

    4. Une fenêtre de navigation s’ouvrira, cliquer sur le bouton de connexion Azure Cloud Login et suivre les invites de commande. Une fois l’authentification Azure terminée, vous serez renvoyé à l’écran ArcGIS Pro.

    Se connecter à une base de données PostgreSQL

    GAE a rationalisé le processus de connexion ArcGIS Pro-PostgreSQL en utilisant une connexion de fichier .sde.

    1. Dans le volet Catalogue, cliquez avec le bouton droit sur Bases de données et sélectionnez Ajouter une base de données propost1

    2. Dans la nouvelle fenêtre, ouvrir Ce PC et localisez votre projet Fileshared (monté en tant que lecteur)

    3. Sélectionnez le fichier .sde, puis cliquez sur Ok

    4. La base de données PostgreSQL n’apparaîtra pas sous Bases de données dans le volet Catalogue.

    Publication sur le portail ArcGIS Enterprise

    En tant que membre sur GAE, vous aurez un rôle d’utilisateur Membre de projet ou de Visionneuse de projet. Les mêmes privilèges seront étendus à ArcGIS Enterprise et à ArcGIS Pro.

    Importation de données à partir d’ArcGIS Enterprise

    Les utilisateurs ayant les rôles Membre du projet et Visionneuse de projet auront la possibilité d’utiliser (afficher) les données partagées de leurs groupes de projets dans ArcGIS Pro ainsi que les données personnelles.

    Dans le volet Catalogue (si le volet Catalogue est fermé, cliquez sur l’onglet Afficher du ruban principal. Dans le groupe Windows, cliquez sur Volet Catalogue), sélectionnez l’onglet Portail.

    Les mêmes Onglets de Contenu d’ArcGIS Enterprise s’afficheront; Mon contenu, Mes favoris, Mes groupes, Mon organisation, ArcGIS Enterprise et Living Atlas.

    Partage de données avec ArcGIS Enterprise

    Seuls les utilisateurs disposant du rôle Membre du projet auront la possibilité de télécharger et de partager du contenu sur le portail.

    Faites un clic droit sur une couche dans la fenêtre Contenu (si la fenêtre Contenu est fermée, cliquez sur l’onglet Afficher du ruban principal.

    Dans le groupe Windows, cliquez sur Contenu) et sélectionnez Partager > Partager en tant que Couche Web dans le menu déroulant. Une nouvelle fenêtre Partager en tant que couche Web apparaîtra.

    Meilleures pratiques

    • Organisez vos données : Créez une structure de dossier logique et utilisez des noms descriptifs pour vos couches et ensembles de données afin de les rendre faciles à trouver et à comprendre.

    • Limitez le nombre de dossiers : Ne créez pas trop de niveaux de dossiers dans votre projet, car cela peut rendre la navigation et la localisation des éléments difficiles. Limitez le nombre de niveaux à pas plus de trois ou quatre.

    • Utilisez les sous-dossiers avec parcimonie : Utilisez les sous-dossiers seulement lorsque c'est nécessaire, par exemple lorsque vous avez un grand nombre d'éléments ou devez les organiser par emplacement ou par thème.

    • Séparez les données de travail des données publiées : Gardez vos données de travail séparées de vos données publiées. Cela peut aider à prévenir la modification ou la suppression accidentelle de données importantes.

    • Utilisez le volet de projet : Utilisez le volet de projet pour gérer vos données, cartes et mises en page. Cela facilite le passage d'une partie de votre projet à une autre.

    • Utilisez les signets : Utilisez les signets pour enregistrer des vues spécifiques de votre carte. Cela facilite le retour à une localisation et un niveau de zoom spécifiques.

    • Utilisez les étiquettes avec parcimonie : Utilisez les étiquettes avec parcimonie et assurez-vous qu'elles sont claires et faciles à lire. Considérez l'utilisation d'un halo ou de remplissages de fond pour faire ressortir les étiquettes.

    • Utilisez la symbologie de manière cohérente : Utilisez une symbologie cohérente sur toutes vos cartes pour les rendre plus faciles à lire et à comprendre.

    • Documentez votre travail : Documentez votre travail en utilisant les outils de métadonnées dans ArcGIS Pro. Cela facilite le partage de votre travail avec les autres et vous aide à suivre les changements au fil du temps.

    • Utilisez les packages de carte : Utilisez les packages de carte pour partager vos cartes et vos données avec les autres. Cela garantit que toutes les données et les paramètres nécessaires sont inclus.

    • Utilisez des géodatabases plutôt que des shapefiles : Globalement, les géodatabases sont un format de stockage de données plus robuste et plus polyvalent que les shapefiles.

    • Optimisez votre flux de travail : Optimisez votre flux de travail en utilisant les outils et les raccourcis disponibles dans ArcGIS Pro. Cela peut vous faire gagner du temps et vous aider à travailler plus efficacement.

    • Profitez des ressources : Profitez de la documentation, de la formation et des communautés d'utilisateurs d'ArcGIS Pro pour en savoir plus sur les meilleures pratiques et obtenir de l'aide lorsque vous en avez besoin.
